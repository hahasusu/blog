CoreDNS 是一个 DNS 服务器。它是用 Go 编写的。

目前默认 CoreDNS 安装中包含大约 30 个插件，但还有一大堆外部插件可以编译到 CoreDNS 中以扩展其功能。

CoreDNS 由插件提供支持

使用 Corefile 来配置 CoreDNS ​

默认情况下会直接监听53端口，并且读取和自己在相同目录下的Corefile配置文件。但是在这种情况下，虽然coredns正常运行了，但是由于没有配置文件，是无法正常解析任何域名请求的。

当CoreDNS启动时，如果没有给出 -conf 标志，它将在当前目录中查找名为 Corefile 的文件。该文件由一个或多个服务器块组成。每个服务器块都会列出一个或多个插件

格式：

bash
ZONE:[PORT] {
	[PLUGIN] ...
}

ZONE : 定义 server 负责的 zone，PORT 是可选项，默认为 53
如果ZONE是 . 则表示有所域名
PLUGIN : 定义 server 所要加载的 plugin。每个 plugin 可以有多个参数

Corefile 中插件的顺序并不决定插件链的顺序。插件的执行顺序由 plugin.cfg 中的顺序决定。

CoreDNS 在其配置中支持环境变量替换。它们可以在 Corefile 中的任何位置使用。语法为 {$ENV_VAR}

将域转发到不同的上游 ​

对 example.org 的查询需要转到 8.8.8.8，其余部分应通过 /etc/resolv.conf 中的名称服务器解析

bash
# 方式一
. {
    forward example.org 8.8.8.8
    forward . /etc/resolv.conf
    log
}

# 方式二
example.org {
    forward . 8.8.8.8
    log
}

. {
    forward . /etc/resolv.conf
    log
}

自定义域名的解析

bash
# vim /etc/coredns/Corefile
test.com {
    file db.test.com
}

# vim /etc/coredns/db.test.com
$TTL 1D
$ORIGIN test.com.  ; 指定 origin，下面的@符号可以作为他的别名，注意后面的 .
@         IN SOA ns.test.com. your.mail.com. (
                20240223   ; serial
                2H         ; refresh (2 hours)
                1H         ; retry (1 hour)
                1D         ; expire (2 weeks)
                1H         ; minimum (1 hour)
                )
          IN    NS    ns.test.com.
          IN     A    10.0.1.0

www       IN     A    10.0.1.0 
db        IN     A    10.0.1.1
doc       IN     A    10.0.1.2
*         IN     A    10.0.1.3  ; 泛解析

# 测试
dig @localhost db.test.com +short # 返回10.0.1.1
dig @localhost doc.test.com +short # 返回10.0.1.2
dig @localhost XXX.test.com +short # 返回10.0.1.3


# 上面的配置只能解析自定义域名，解析其它域名添加 forward
. {
    forward . 223.5.5.5
}

脚本 ​

bash
wget https://github.com/coredns/coredns/releases/download/v1.11.1/coredns_1.11.1_linux_amd64.tgz
tar xf coredns_1.11.1_linux_amd64.tgz -C /usr/local/bin



domain=test.com
mkdir /etc/coredns
# 1 Corefile
cat >  /etc/coredns/Corefile << EOF
$domain {
    file /etc/coredns/db.$domain
}

. {
    forward . 223.5.5.5
}
EOF

# 2 区域文件
cat > /etc/coredns/db.$domain <<EOF
\$TTL 1D
\$ORIGIN test.com.  
@         IN SOA ns.${domain}. your.mail.com. (
                20240223   ; serial
                2H         ; refresh (2 hours)
                1H         ; retry (1 hour)
                1D         ; expire (2 weeks)
                1H         ; minimum (1 hour)
                )
          IN    NS    ns.${domain}.
          IN     A    10.0.1.0

www       IN     A    10.0.1.0 
db        IN     A    10.0.1.1
doc       IN     A    10.0.1.2
*         IN     A    10.0.1.3  ; 泛解析
EOF

# 3 service 文件
cat > /lib/systemd/system/coredns.service <<EOF
[Unit]
Description=CoreDNS
Documentation=https://coredns.io/manual/toc/
After=network.target

[Service]
Type=simple
User=root
# 指定运行端口和读取的配置文件
ExecStart=/usr/local/bin/coredns -conf /etc/coredns/Corefile
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

echo run systemctl enable --now coredns.service
