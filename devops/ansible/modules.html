常用模块 ​

可以从ansible-doc -l | grep 来找出想要的模块。再使⽤ansible-doc -s module_name来查看此模块的⽤法

ansible绝⼤多数模块都天然具有幂等特性，只有极少数模块如shell和command模块不具备幂等性。所谓的幂等性 是指多次执⾏同⼀个操作不会影响最终结果。
例如，ansible的yum模块安装rpm包时，如果待安装的包已经安装过 了，则再次或多次执⾏安装操作都不会真正的执⾏下去。
再例如，copy模块拷贝⽂件时，如果⽬标主机上已经有了 完全相同的⽂件，则多次执⾏copy模块不会真正的拷贝。ansible具有幂等性的模块在执⾏时，都会⾃动判断是否 要执⾏。

1 shell 和 command ​

默认ansible使⽤的模块是command，即可以执⾏⼀些shell命令。shell和command的⽤法基本⼀样，实际上shell 模块执⾏命令的⽅式是在远程使⽤/bin/sh来执⾏的，如/bin/sh ping

bash
ansible-doc -s shell
- name: Execute shell commands on targets
  shell:
    chdir: # 在运行命令之前更改到此目录。
    cmd: # 要运行的命令，后跟可选参数。
    create: # 一个文件名，当它已经存在时，这一步将*不会*运行。
    executable: # 更改用于执行命令的 shell。 这需要可执行文件的绝对路径。
    free_form: # shell 模块采用自由格式命令作为字符串运行。 没有名为“自由形式”的实际参数。 请参阅
    removes: # 一个文件名，当它不存在时，这一步将*不*运行。
    stdin: # 将命令的stdin直接设置为指定值。
    stdin_add_newline: # 是否在标准输入数据中追加换行符

在ansible中使⽤shell或command模块⼀定要注意，它们默认不满⾜幂等性，很多操作会重复执⾏，但有些操作是 不允许重复执⾏的。例如mysql的初始化命令mysql_install_db，它只能在第⼀次配置的过程中初始化⼀次，其他任 何时候如⾮需要则不允许执⾏。这时候要实现幂等性，可以通过模块的creates和removes选项进⾏判断，但⽆论 如何，在执⾏这两个模块的时候都需要考虑要执⾏的命令是否应该实现幂等性。

unarchive ​

功能：解包解压缩

  1. 将 ansible 主机上的压缩包传到目标主机后解压至特定目录
  2. 将其它主机的压缩包解压到目标主机指定目录，需设置remote_ser=yes
bash
ansible all -m unarchive -a 'src=https://nginx.org/download/nginx-1.24.0.tar.gz dest=/tmp remote_src=yes'
ansible all -m unarchive -a 'src=/tmp dest=/tmp owner=luo group=luo'

bash
src
要复制到远程服务器的文件的本地路径。
这可以是绝对的或相对的。
如果path是目录，则递归复制。在这种情况下，如果路径以“/”结尾，则仅将该目录的内部内容复制到目标。否则，如果不以“/”结尾，则复制目录本身及其所有内容。此行为类似于 rsync 命令行工具。
