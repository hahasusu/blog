  • libssl：用于实现ssl通信协议的安全库
  • openssl：多用途命令行工具

实现私有CA和证书申请 ​

  1. 生成证书申请请求
  2. RA核验
  3. CA签署
  4. 获取证书

openssl的配置文件: /etc/pki/tls/openssl.cnf

三种策略:

bash
match：要求申请填写的信息跟CA设置信息必须一致
optional：可有可无，跟CA设置信息可不一致
supplied：必须填写这项申请信息
1
2
3

1. 创建CA所需要的文件

bash
mkdir -p /etc/pki/CA/{certs,crl,newcerts,private}

#生成证书索引数据库文件
touch /etc/pki/CA/index.txt
#指定第一个颁发证书的序列号
echo 01 > /etc/pki/CA/serial

2. 创建CA的私钥
cd /etc/pki/CA/
# umask 066 # 新版本不用设置权限
openssl genrsa -out private/cakey.pem #  2048(默认值，可不写)


3. 给CA颁发自签名证书
openssl req -new -x509 -key /etc/pki/CA/private/cakey.pem -days 3650 -out /etc/pki/CA/cacert.pem

# 说明
-new：生成新证书签署请求
-x509：专用于CA生成自签证书
-key：生成请求时用到的私钥文件
-days n：证书的有效期限
-out /PATH/TO/SOMECERTFILE: 证书的保存路径

方式二 ​

创建CA ​

bash
domain=test.com

# 生成 CA 证书私钥
openssl genrsa -out ca.key 4096

# 生成 CA 证书
openssl req -x509 -new -nodes -sha512 -days 3650 \
 -subj "/C=CN/ST=Beijing/L=Beijing/O=example/OU=dev/CN=${domain}" \
 -key ca.key \
 -out ca.crt

签发证书 ​

bash
 # 1 生成私钥
 openssl genrsa -out ${domain}.key 4096
 
 # 2 生成证书签名请求 (CSR)
 openssl req -sha512 -new -key ${domain}.key -out ${domain}.csr \
    -subj "/C=CN/ST=Chengdu/L=Chengdu/O=test/OU=dev/CN=${domain}"
    

# 3 使用ca签发证书
openssl x509 -req -sha512 -days 3650 \
    -CA ca.crt -CAkey ca.key -CAcreateserial \
    -in ${domain}.csr \
    -out ${domain}.crt
